ℹ Automation vs Manual — Classification Guide

Automated

  • Runs repeatedly (regression, CI/CD)
  • Has deterministic, binary pass/fail criteria
  • Data-driven or parameterised
  • API / integration / unit tests
  • Login, form validation, CRUD operations
  • Performance benchmarks & load tests

Example: "Verify that POST /api/users returns 201 and the correct JSON body when given valid data."

Manual

  • Requires human judgement or perception
  • UI/UX, visual design, or accessibility review
  • Exploratory or ad-hoc sessions
  • One-time or rarely repeated scenarios
  • Involves physical hardware or external devices
  • Subjective outcome (e.g. "feels intuitive")

Example: "Explore the onboarding flow as a first-time user and note any points of confusion."
